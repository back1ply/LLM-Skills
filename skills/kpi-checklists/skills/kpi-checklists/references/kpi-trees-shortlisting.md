# KPI Trees & Shortlisting

## KPI Trees — Generating Your Longlist

### What Is a KPI Tree?

A KPI Tree is a graphical tool showing linkages between strategic objectives and everyday measures. It decomposes top-down:

```
Strategic Objective
  └── Theme (broad area)
        └── Tactical (specific enabler)
              └── Measure (what you actually track)
```

### Why Use KPI Trees?

1. **Sum up complexity** — Sub-measures group naturally into meaningful summaries
2. **Build agreement** — Collaborative workshops prevent dominant characters from railroading decisions
3. **Explain the approach** — Takes 5 minutes to explain how to read one; powerful for executive strategy communication
4. **Keep pace with strategy changes** — See impact of strategy changes on measures
5. **Understand measure interactions** — Expose when pushing one "good" measure causes negative side effects

### The Three Link Types

| Link Type | Visual | Description | Example |
|-----------|--------|-------------|---------|
| **Cause-effect** | Grey line | One activity directly influences another | "Sell popular flavours" causes "Increase in ice cream sales" |
| **Conflict** | Red line | Activities work against each other | "Minimise AHT" conflicts with "Maximise First Touch Resolution" |
| **Companion** | Double line | Overlap or subset relationship | "Weight" is a companion to "Body Mass Index" |

### The AHT Warning

Average Handling Time (AHT) in call centres is a classic example of measure interaction. Pushing AHT reduction can:
- Increase call volumes (agents fob off callers)
- Decrease first touch resolution
- Plummet customer satisfaction
- While AHT goes down, complaints triple

Always check: "Is there a way of making a measure go the 'right' way by doing something stupid?"

## Building KPI Trees — Workshop Process

### Preparation Checklist

- [ ] Become fully familiar with the strategy and strategic objectives
- [ ] Double-check objectives with all engaged senior stakeholders — differences must be ironed out before sessions
- [ ] Check 2-7 strategic objectives (more suggests lower-level tactical objectives are mixed in)
- [ ] Gain support and approval from senior stakeholder(s) to hold sessions
- [ ] Gather example outputs from previous sessions (or use book examples)
- [ ] Select groups of 3-9 people (no more than 12)
- [ ] Good mix of seniority in each group
- [ ] Organise two 2-hour workshops, separated by 1-5 working days

### Practicalities

- [ ] Book meeting rooms for both sessions
- [ ] Create briefing email and send invitations
- [ ] Base group selection around broadly similar remits
- [ ] Ensure desks are available to work at
- [ ] Whiteboard available if possible
- [ ] Print out examples and worksheets
- [ ] Take Post-Its, pens, and a camera phone

### Session 1 Checklist

- [ ] Explain the approach
- [ ] Identify strategic objectives — agree with the group
- [ ] Give background to the session
- [ ] Show a finished example
- [ ] Get the group to do a simple (non-work) exercise example
- [ ] Explain the three link types: cause-effect, conflict, and companion
- [ ] Help group work through a more complex non-work example including link types
- [ ] Get the group to develop a draft KPI Tree for their relevant strategic objectives
- [ ] Develop one tree per objective (trees will almost certainly cross-link)

**Reassurance**: If participants get anxious about the number of potential measures, reassure them that the longlist will be shortlisted in the next step.

### Follow-Up After Session 1

- [ ] Write up trees from all groups and merge into one tree
- [ ] Add notes showing where merge decisions were made

### Session 2 Checklist

- [ ] Review the merged tree
- [ ] Are there important factors that won't register with any of the identified measures?
- [ ] Is there a way of making a measure go the "right" way by doing something stupid?
- [ ] Add any further branches needed
- [ ] Make corrections and discuss the merged tree

### Follow-Up After Session 2

- [ ] Finish drawing up trees from the second session
- [ ] There should be only one version per objective (merged after session 1)
- [ ] Add explanatory notes as needed
- [ ] Circulate to participants for final approval
- [ ] State that absence of a response = implicit approval

### Tools for Drawing KPI Trees

**Wire Tree diagrams** (most space-efficient, 5-10x fewer pages than org-chart style):
- Can be built entirely in Excel using the Borders function
- Allows adding supporting data in cells off to the right

**Mind Map style tools**:
- Visio (PC)
- Word/PowerPoint Draw Tools (PC, Mac)
- Omnigraffle (Mac)
- SmartDraw (PC)
- Lucid (Web app, multi-platform)
- MindMeister (Web app, free)
- Mindjet MindManager (Mac, PC)
- Xmind (PC, Mac, free)
- FreeMind (Java, free)

## Step 4 — Shortlisting KPIs

### The Shortlisting Matrix

Rate each candidate measure (lowest-level boxes on KPI Tree only) on:

| Criterion | Scale | What 10 means | What 0 means |
|-----------|-------|----------------|--------------|
| **Importance** | 0-10 | Critical to the business | Irrelevant |
| **Availability** | 0-10 | Auto-generated, fully trusted, sliceable with no effort | Effectively unavailable, unrealistic effort to collect |

### Process Checklist

- [ ] Make sure you're dealing with actual measures (lowest-level KPI Tree boxes), not higher-level headings
- [ ] Get a flip chart
- [ ] Label Importance on the vertical axis, Availability on the horizontal
- [ ] Number the bottom-level measures on the KPI Tree for reference
- [ ] Add dividing lines at midpoint of each axis (4-box grid)
- [ ] Go through each measure with the group, rating Importance and Availability
- [ ] Note down any disagreements about scores (useful audit trail)
- [ ] Plot each measure with a cross and the measure name/number

**Tip**: Use Post-it notes for flexibility in debate. Take a photo before removing from the flip chart!

### How to Use the Matrix

| Quadrant | Description | Action |
|----------|-------------|--------|
| **Top Right (Use)** | High importance + easy to collect | Put on first dashboard/report |
| **Top Left (Aspire)** | High importance + hard to collect | Develop these — your KPI development to-do list |
| **Bottom Right (Caution)** | Low importance + easy to collect | Don't include just because you can — causes report bloat |
| **Bottom Left (Forget)** | Low importance + hard to collect | Park these with rationale explaining why |

### The Importance of the Discard List

Always record rejected measures with reasons. Benefits:
- "We didn't go with that measure and here are the reasons why" is powerful for preventing report bloat
- Demonstrates the rigour of the selection process
- Defends choices against future challenges

### Developing Aspire List Measures

Set each as a mini-project:

- [ ] Confirm you really want to measure this
- [ ] Identify stakeholders using RACI matrix
- [ ] Define the measure using KPI definition sheet
- [ ] Review definition with key stakeholders
- [ ] Do a quick feasibility study (especially if IT is involved)
- [ ] Check organisational project management requirements
- [ ] Assign a project manager (full-time or part-time)
- [ ] Put together a one-page charter (objectives, resources, timescales, issues)
- [ ] Draft a simple project plan
- [ ] Review timescale and cost — still want to go ahead?
- [ ] Create and maintain an action log
- [ ] Set up simple project governance

### Action Log Fields

- [ ] Unique reference number
- [ ] Grouping description (e.g., "IT", "Training")
- [ ] Meaningful description of the action
- [ ] Who owns the action
- [ ] When it is due
- [ ] When it is closed
- [ ] Whether on hold
- [ ] Relevant notes or comments
- [ ] Risks (optional)
- [ ] Mitigations (optional)

### Measuring the Unmeasurable

When you can't measure something directly:
- Look at something that **affects** it, or
- Look at something that is **affected by** it

Example: Paper thickness can't be measured without stopping the process, so measure gamma radiation absorption rate instead.

Example: Toddler interest in TV measured by frequency of distraction by a pile of toys (used by Sesame Street creators).
